VENDOR TABLES MAPPING - COMPLETE INTEGRATION
=============================================

All vendor* tables are now properly mapped and integrated in dataEntryApi.js

VENDOR TABLE METHODS IMPLEMENTED:
=================================

1. vendor_documents (KYC Documents)
   - getVendorDocuments(vendorId) → Get all KYC documents
   - addVendorDocument(vendorId, type, url, name) → Add new document
   - Used in: KycApprovals.jsx (displaying vendor KYC docs)

2. vendor_bank_details
   - getVendorBankDetails(vendorId) → Get all bank accounts
   - addBankAccount(vendorId, bankData) → Add new bank account
   - Fields: account_number, account_holder, bank_name, ifsc_code, branch_name, is_primary

3. vendor_contact_persons
   - getVendorContacts(vendorId) → Get all contact persons
   - addContactPerson(vendorId, contactData) → Add new contact
   - Fields: name, designation, mobile_number, email, is_primary

4. vendor_preferences
   - getVendorPreferences(vendorId) → Get vendor's preferred categories/locations
   - setVendorPreferences(vendorId, preferences) → Set/update preferences
   - Fields: preferred_micro_categories, preferred_states, preferred_cities, min_budget, max_budget, auto_lead_filter

5. vendor_payments
   - getVendorPayments(vendorId) → Get payment history
   - recordVendorPayment(vendorId, paymentData) → Record new payment
   - Fields: amount, description, status, payment_method, transaction_id, invoice_url

6. vendor_messages
   - getVendorMessages(vendorId) → Get all messages for vendor
   - Fields: sender_name, sender_email, sender_phone, subject, message, is_read, is_replied

7. vendor_lead_quota
   - getVendorLeadQuota(vendorId) → Get lead quota info
   - Fields: daily_used, daily_limit, weekly_used, weekly_limit, yearly_used, yearly_limit

8. vendor_plan_subscriptions
   - getVendorSubscriptions(vendorId) → Get active subscriptions
   - Joins with: vendor_plans table

9. vendor_plans
   - getVendorPlans() → Get all active plans
   - Fields: name, daily_limit, weekly_limit, yearly_limit, member_limit, price, features

10. vendor_additional_leads
    - getAdditionalLeads(vendorId) → Get additional leads purchased
    - Fields: leads_purchased, leads_remaining, purchase_date, expiry_date, amount_paid

11. vendor_subscriptions (Services)
    - getVendorServiceSubscriptions(vendorId) → Get service subscriptions
    - Joins with: vendor_services table

12. vendor_otp_codes
    - createVendorOTP(vendorId, email) → Create OTP for vendor verification
    - Auto-generates 6-digit code, expires in 10 minutes

KEY INTEGRATION POINTS:
=======================
- All methods follow consistent error handling pattern
- All return empty arrays [] or null for not found cases
- All methods use Supabase from 'vendor_*' tables
- All vendor data accessed via vendor_id foreign key
- Realtime capable (can be wrapped with subscriptions)

USAGE IN COMPONENTS:
====================
KycApprovals.jsx → Uses getVendorDocuments() to fetch KYC documents
VendorOnboarding.jsx → Can use addBankAccount(), addContactPerson()
Future Vendor Dashboard → Can use all metrics methods for vendor analytics

NO UNUSED TABLES:

All vendor* tables in schema are mapped and have corresponding methods.
